# Image 2 Local Studio 使用说明

这是一个**纯开源、纯静态、无中间服务器**的图片工具。

你可以把它理解成：

- 一个本地浏览器里的图片工作台
- 一个帮你更方便调用中转接口的前端页面
- 一个适合买了中转、但嫌自己直接调接口太麻烦的人使用的小工具

整个项目核心就是一个 [index.html](./index.html)。

它**不会把你的请求先发到作者服务器再转发**。  
接口地址和 Token 都是你自己填，浏览器直接请求你自己的接口提供方。

所以它的特点很明确：

- 开源
- 透明
- 没有中间服务器
- 不替你托管接口
- 不替你存图片
- 安全性主要取决于你自己的机器和你填写的接口提供方

---

## 这工具适合谁

适合这些人：

- 已经买了中转接口
- 手里有 `API 地址` 和 `Token`
- 但不想自己写代码调接口
- 觉得 Postman、脚本、SDK 太麻烦
- 想直接点点页面就生成图片

也适合：

- 想体验文生图 / 图生图的人
- 想看历史记录、导出图片的人
- 想让提示词助手帮忙整理提示词的人

## 这个工具能做什么

### 1. 文生图

你输入提示词，直接生成图片。

适合：

- 从零开始出图
- 海报
- 角色图
- 室内场景
- 产品图

### 2. 图生图 / 局部改图

你可以上传参考图，再让模型继续生成或局部调整。

支持：

- 上传参考图
- 上传蒙版
- 从历史图片里直接拿一张继续改

### 3. 提示词助手

右侧有提示词协作台。

它会参考当前：

- 模式
- 尺寸
- 质量
- 响应格式
- 是否有参考图
- 是否有蒙版
- 最近选择的模板

然后帮你把自然语言整理成更适合提交的提示词。

### 4. 本地历史

历史图片保存在浏览器本地缓存（IndexedDB）里。

支持：

- 搜索
- 分页
- 删除指定缓存
- 清空全部缓存
- 导出图片
- 继续作为图生图参考

## 使用前你需要准备什么

你只需要准备两样：

1. **接口地址**
2. **Token**

也就是你买中转之后，商家一般都会给你的这两项。

如果你用的是当前页面默认兼容提供方，图片生成地址默认可以理解为：

`https://ruoli.dev/v1/images/generations`

如果你的中转平台不是这个地址，也没关系，页面配置里现在支持手动修改“图片生成地址（可选）”。

如果你还没有可用接口，页面配置区也留了一个可选入口：

https://ruoli.dev/register?aff=8GLe

## 怎么开始用

### 第一步：打开页面

直接打开：

- `index.html`

推荐用 Edge 或 Chrome。

### 第二步：点右上角“连接设置”

把你买到的：

- API 地址
- Token

填进去。

### 第三步：测试一下

建议先点：

- `测试连接`
- `测试图片模型`

这样可以先确认：

- 地址是不是对的
- Token 能不能用
- 你的中转是否真的开了图片模型

### 第四步：选择模式

顶部导航里可以切：

- `文生图`
- `图生图 / 局部改图`
- `配置说明`

#### 文生图

适合从零开始。

#### 图生图 / 局部改图

适合拿已有图片继续改。

### 第五步：输入提示词并生成

你可以：

- 自己写提示词
- 用右侧提示词助手帮你整理

然后点击：

- `开始生成`

## 这个工具安全吗

从结构上说，它是相对透明的。

因为它是：

- 纯前端
- 纯静态
- 开源
- 没有中间服务器

也就是说：

- 你的图片请求不会先绕到作者服务器
- 你的地址和 Token 是浏览器直接拿去请求你填的接口提供方

但你也要知道：

### 1. Token 仍然在你浏览器本地保存

为了方便使用，配置会保存在浏览器本地。

所以建议：

- 只在你信任的机器上使用
- 不要在公共电脑上长期保存

### 2. 安全还取决于你的接口提供方

这个工具本身不做二次转发。  
你填的接口是谁，就由谁接收请求。

---

## 图片历史保存在哪里

图片历史默认保存在：

- 浏览器本地缓存（IndexedDB）

这意味着：

- 它不在项目文件夹里
- 不是系统图片目录
- 你在资源管理器里不会直接看到这些历史图

如果你想把它们真正保存成文件：

- 用页面里的导出功能

如果你不想留缓存：

- 删除指定历史
- 或清空全部缓存

---

## 关于尺寸、质量、4K、全景图

页面里已经内置了一些常用尺寸，也支持你尝试更大的尺寸或自定义尺寸。

但这里有个很重要的点：

**不是页面能选，就一定代表你的接口提供方真的支持。**

比如：

- 4K
- 自定义尺寸
- 360 全景尝试

很多时候取决于：

- 你的中转平台有没有扩展支持
- 它后端接的模型能力是不是放开了

所以可以这样理解：

- 页面负责给你入口和体验
- 真能不能生成，还是看你买的接口

## 如果遇到问题，先看这几个点

### 1. 测试连接失败

先检查：

- API 地址有没有填错
- Token 有没有复制错
- Token 是否过期

### 2. 图片模型测试失败

一般优先怀疑：

- 你买的接口没开图片模型
- 地址虽然可用，但不是图片接口
- 该中转限制了某些模型或参数

### 3. 页面能打开，但生成不出来

常见原因：

- 上游接口不允许浏览器直连
- 跨域被拦
- 你当前选的尺寸/参数该中转不支持

### 4. 历史图片能看但导不出来

常见原因：

- 某些记录只是远程 URL
- 目标服务本身限制了再次抓取

## 已知边界

这些不一定是项目 bug，更像是使用边界：

### 1. 浏览器跨域限制

这是纯前端工具。  
如果你的上游接口不允许浏览器直接访问，那页面也没法绕过去。

### 2. 不同中转支持能力不同

同样的界面参数：

- 有的中转支持
- 有的不支持

尤其是：

- 4K
- 自定义尺寸
- 图生图
- 蒙版局部改图

### 3. 提示词助手依赖文本模型

图片能生成，不代表提示词助手一定能用。  
右侧助手通常还依赖你的中转同时支持文本模型能力。
